# 项目记忆 - Baby v0.1

## 开发模式约定

本项目采用 **AI 多会话虚拟团队协作开发模式**，核心规则：

### 虚拟角色分工
- **R0. PM 项目经理**：全局统筹，拆解需求、编写标准任务卡、分配任务、验收成果、维护项目状态
- **R01. 架构师**：技术架构、技术选型、模块划分、接口契约、全局技术规范，不写业务代码
- **R02. 后端工程师**：实现业务代码、数据层、接口逻辑，不修改架构定义
- **R03. QA 测试工程师**：编写用例、全场景测试、输出 bug 清单，不新增业务功能

### 任务卡规范（PM 下发必须包含以下全部字段）

```
# 标准任务卡模板
任务ID：
对应项目模块：
🎯 完成目标：
📋 允许修改范围（文件/模块）：
禁止改动范围：
✅ 验收标准（逐条可验证）：
🔖 代码分支命名规则：
📄 完工汇报必须包含板块：
1.任务卡引用 2.分支名 3.修改文件清单 4.改动说明 5.验收自查结果 6.遗留问题 7.交付产物
```

任何下发任务卡缺少以上任一字段，执行角色应拒绝开工，要求 PM 补全。

### 闭环协作流程
PM 编写任务卡 → 分配角色 → 角色执行 → 结构化回报 → PM 验收 → 合格放行/打回返工 → 更新全局文档

### 结构化回报模板（工程师/架构/QA 完工必用）
1. 任务 ID 与任务卡原文引用
2. 当前代码分支名称
3. 修改文件清单
4. 核心改动逻辑说明
5. 验收标准逐条自查（达标/未达标）
6. 现存缺陷、待解决问题
7. 交付产物

### 核心约束
- 每条会话仅承担一个岗位，禁止跨岗操作
- 项目真相以本地仓库文件为唯一基准
- 禁止随意扩大开发范围，严格遵循任务卡边界
