## 📋 允许修改范围

### 你可以做的：
- ✅ 在已确认的技术栈内做细化选型（如 CSS 方案、路由方案、HTTP 客户端）

- ✅ 定义 Vercel Functions 的 API 接口契约（路径、入参、出参、错误码）
- ✅ 设计 Pinia Store 结构（持仓数据、基金数据、UI 状态）
- ✅ 设计 Service Worker 缓存策略（哪些资源预缓存、哪些运行时缓存、缓存过期策略）
- ✅ 确定 Vue Router 路由表（页面列表、路由参数）
- ✅ 确定 ECharts 图表组件的数据接入格式
- ✅ 补充你认为必要的技术决策和边界定义
- ✅ 新增 `ai/memory-bank/` 下的架构设计文档

### 你不能做的：
- ❌ 不写业务代码（不实现任何 .vue / .js / API 函数）
- ❌ 不修改项目章程（project-charter.md）
- ❌ 不写测试用例（那是 R03 的职责）
- ❌ 不改变已确认的技术栈（Vue 3 + Vite + Vercel Functions + ECharts）

## ✅ 验收标准

| # | 标准 | 判定方式 |
|---|------|---------|
| 1 | 输出**项目目录结构**，包含前端和 API 两个子项目的完整文件树 | R0 检查 |
| 2 | 输出**Vercel Functions API 接口契约**，每条 API 包含：路径、请求方法、Query/Body 参数、成功响应 JSON 示例、错误响应格式 | R0 逐条检查 |
| 3 | 输出**Vue 路由表**，列出所有页面路径、页面组件、路由参数 | R0 检查 |
| 4 | 输出**Pinia Store 结构设计**，每个 Store 的名称、state/actions/getters 定义 | R0 检查 |
| 5 | 输出**组件树**，至少覆盖 P0 的 4 个页面功能（持仓总览、搜索添加、盈亏详情、PWA 壳） | R0 检查 |
| 6 | 输出**Service Worker 缓存策略**，明确预缓存列表和运行时缓存规则 | R0 检查 |
| 7 | 所有接口设计必须**覆盖全部 4 个数据源**（天天基金估值、东方财富历史净值、基金搜索、大盘指数） | R0 逐条对照 |
| 8 | 考虑**非交易时段**：估值接口盘后无数据时的降级策略 | R0 检查 |
| 9 | 考虑**错误处理**：API 超时、返回异常时的前端展示策略 | R0 检查 |
| 10 | 架构文档保存到 `ai/memory-bank/architecture.md` | R0 检查 |
