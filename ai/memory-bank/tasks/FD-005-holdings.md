## ✅ 验收标准

| # | 标准 | 判定方式 |
|---|------|---------|
| 1 | 页面路由 `/` 正确渲染 `HoldingsOverview.vue` | 浏览器访问 `/` |
| 2 | **首次加载（无持仓）**：显示 `EmptyState` 组件，文案 "还没有添加基金"，按钮 "去添加" 跳转到 `/search` | 清除 localStorage 后刷新 |
| 3 | **加载中**：显示 `SkeletonLoader` 骨架屏（模拟 FundCard 占位） | 添加 setTimeout 模拟 |
| 4 | **有持仓数据**：以 `FundCard` 列表展示每只基金，包含基金名称、代码、估值/净值、涨跌幅 | 手动写入 localStorage |
| 5 | **盘中时段**：显示实时估值 + "实时" 标签（绿色脉冲动画，参考架构行 667） | 调系统时间验证 |
| 6 | **盘后/非交易日**：显示昨日净值 + "盘后" 灰色标签，隐藏估值涨跌幅 | 调系统时间验证 |
| 7 | **离线状态**：显示缓存数据 + 顶部 NetworkStatus 黄色提示条，不中断浏览 | 断网测试 |
| 8 | **加载失败 + 有缓存**：显示上次数据 + 顶部黄色提示 "数据更新失败，显示上次数据" | mock API 报错测试 |
| 9 | **加载失败 + 无缓存**：显示 `ErrorState` + "重试" 按钮 | mock API 报错 + 清缓存 |
| 10 | 调用 `useAutoRefresh(store.fetchValuations, 30000)` 实现交易时段自动刷新 | 检查代码 |
| 11 | 页面 `onMounted` 时从 localStorage 加载持仓并拉取估值 | 检查生命周期 |
| 12 | 底部 `NavBar` 的 "持仓" Tab 处于高亮状态 | 视觉检查 |
| 13 | 每张 FundCard 点击跳转到 `/fund/:code` 详情页 | 点击测试 |
| 14 | 页面顶部显示总资产、总盈亏汇总数字（依赖 useFundStore getters） | 视觉检查 |
| 15 | 移动端 375px 宽度下卡片布局正常，无横向溢出 | DevTools 模拟 |
