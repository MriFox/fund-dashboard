## ✅ 验收标准

| # | 标准 | 判定方式 |
|---|------|---------|
| 1 | 页面路由 `/search` 正确渲染 `FundSearch.vue` | 浏览器访问 `/search` |
| 2 | 页面顶部有搜索输入框，placeholder 为 "输入基金代码或名称" | 视觉检查 |
| 3 | 搜索输入框实现 **300ms 防抖**，用户停止输入 300ms 后才发起请求 | 检查防抖逻辑 |
| 4 | 输入基金代码（如 `110022`）搜索，展示匹配的基金列表（FundCard + "添加" 按钮） | 输入测试 |
| 5 | 输入基金名称（如 "易方达"）搜索，展示模糊匹配结果 | 输入测试 |
| 6 | **搜索中**：显示 SkeletonLoader 骨架屏 | 慢网络模拟 |
| 7 | **无匹配结果**：显示 EmptyState "未找到相关基金" | 输入乱码测试 |
| 8 | **搜索失败**：显示 ErrorState + 重试按钮 | mock API 报错 |
| 9 | 点击某只基金的 "添加" 按钮，弹窗要求输入：**持有份额**、**成本净值**、**买入日期** | 视觉 + 交互测试 |
| 10 | 提交添加表单后，数据写入 localStorage（通过 useFundStore.addFund），跳转到 `/` | 检查 localStorage |
| 11 | 已添加的基金在搜索结果中标记 "已添加"，不可重复添加 | 重复搜索测试 |
| 12 | 底部 NavBar 的 "添加" Tab 处于高亮状态 | 视觉检查 |
| 13 | 搜索输入框按回车键触发搜索（非防抖，立即搜索） | 键盘测试 |
| 14 | 移动端 375px 宽度下布局正常 | DevTools 模拟 |
