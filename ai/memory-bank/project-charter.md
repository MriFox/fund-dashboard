# 个人基金看板 PWA — 项目章程

> 版本 v1.0 | 2026-07-07 | R0 PM

---

## 一、项目定义

| 项 | 内容 |
|----|------|
| **项目名称** | 个人基金看板 PWA |
| **项目代号** | fund-dashboard |
| **项目类型** | PWA 移动端 Web 应用 |
| **目标用户** | 1 人（个人自用） |
| **核心目标** | 在手机上随时查看持有基金的实时估值、盈亏、走势，可安装到桌面 |
| **参考产品** | 养基宝（取其核心功能，自主设计 UI） |
| **预算约束** | 0 元（全部使用免费服务） |

---

## 二、技术栈（已确认）

| 层 | 选型 | 理由 |
|----|------|------|
| **前端框架** | Vue 3 + Vite | PWA 插件成熟、ECharts 官方封装、中文生态好 |
| **PWA** | vite-plugin-pwa | 开箱即用 Service Worker + Manifest |
| **图表** | ECharts + vue-echarts | 折线/饼图/K 线，免费 |
| **状态管理** | Pinia | Vue 3 官方推荐 |
| **CSS** | 待架构师确定（Tailwind / UnoCSS / 纯 CSS） | |
| **后端/代理** | Vercel Serverless Functions | 免费额度远超需求，天然 CORS 代理 |
| **数据存储** | 前端 localStorage + Vercel Edge Config（可选） | 用户持仓数据存本地 |
| **部署** | Vercel（前端 + API 同仓库） | 免费、自动 CI/CD |
| **代码仓库** | GitHub（私有仓库） | 免费 |

## 三、数据源

| 接口 | 来源 | 用途 |
|------|------|------|
| `fundgz.1234567.com.cn/js/{code}.js` | 天天基金 | 实时估值、估算涨跌幅 |
| `api.fund.eastmoney.com/f10/lsjz` | 东方财富 | 历史净值（走势图） |
| `fund.eastmoney.com/js/fundcode_search.js` | 东方财富 | 全市场基金搜索 |
| `push2.eastmoney.com/api/qt/ulist.np/get` | 东方财富 | 大盘指数 |

> 以上均为公开接口，个人自用无需授权。

## 四、MVP 范围（三阶段）

### P0 — 第一轮交付（核心可用）

| # | 功能 | 说明 |
|---|------|------|
| P0-1 | PWA 壳 | 可安装到手机桌面、启动页、离线缓存、Service Worker |
| P0-2 | 持仓总览 | 基金列表 + 实时估值 + 今日涨跌 + 持有收益 |
| P0-3 | 基金搜索/添加 | 输入代码或名称搜索，添加到我的持仓 |
| P0-4 | 持仓盈亏详情 | 录入份额和成本价，自动算持有收益、收益率 |
| P0-5 | Vercel 数据代理层 | Serverless Functions 代理所有外部基金 API |

### P1 — 第二轮迭代（可视化增强）

| # | 功能 | 说明 |
|---|------|------|
| P1-1 | 净值走势图 | 近 30 天 / 近 1 年 ECharts 折线图 |
| P1-2 | 资产配置饼图 | 按基金类型分布的环形图 |
| P1-3 | 大盘指数卡片 | 上证/深证/创业板实时行情 |
| P1-4 | 自动刷新 | 每 30 秒拉取估值，非交易时段免请求 |

### P2 — 第三轮（功能深度）

| # | 功能 | 说明 |
|---|------|------|
| P2-1 | 基金对比 | 选 2-3 只基金同屏对比收益率走势 |
| P2-2 | 持仓收益曲线 | 持有以来的累计收益率曲线 |
| P2-3 | 基金详情页 | 历史业绩、最大回撤、基金经理信息 |

## 六、开发流程

```
PM编写任务卡 → 分配角色 → 角色执行 → 结构化回报 → PM验收 → 更新状态
```

- 每条任务卡必须包含 5 要素：🎯目标 / 📋修改边界 / ✅验收标准 / 🔖分支命名 / 📄回报格式
- 角色完工必须使用结构化回报模板
- 项目真相以本地仓库文件为准

---

## 七、非功能需求

- 移动端优先设计（375px 基准）
- 离线可查看缓存数据（Service Worker 策略）
- API 请求失败时有降级展示
- 红涨绿跌（或可配置）
- 加载状态和骨架屏
